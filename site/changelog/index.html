    Migration Guide

    From v0.1.0 to v0.2.0 (When Released)

    Database Migration: The v0.2.0 release will include automatic database migration for existing users:

    # Backup your current database (recommended)
cp ~/.imos/memory.db ~/.imos/memory.db.backup

# Upgrade IMOS
pip install --upgrade imos

# Migration runs automatically on first use
imos list

    New Dependencies: v0.2.0 will require additional Python packages: - pandas>=1.3.0 (for CSV/Excel support) - markdown>=3.4.0 (for Markdown import) - beautifulsoup4>=4.9.0 (for HTML processing)

    Breaking Changes: - None planned for v0.2.0 (maintaining backward compatibility)

    Known Issues

    v0.1.0 Current Issues

    1. Large PDF Processing: Files >50MB may cause memory issues
    2. Workaround: Split large PDFs or use text extraction tools first

    3. Fix planned: v0.2.0 streaming PDF processor

    4. API Rate Limits: Heavy chat usage may hit Groq API limits

    5. Workaround: Use chat moderately or provide your own API key

    6. Fix planned: v0.2.1 intelligent rate limiting

    7. Memory Search Order: Results sometimes appear in unexpected order

    8. Workaround: Use more specific search terms

    9. Fix planned: v0.2.0 improved ranking algorithm

    10. Database Location: Database created in current directory initially

    11. Workaround: Database moves to user home on first use

    12. Fix planned: v0.1.1 patch for proper initial location

    13. Windows Path Handling: Some path characters may cause issues

    14. Workaround: Avoid special characters in file paths
    15. Fix planned: v0.1.1 improved path normalization

    Reporting Issues

    Found a bug? Help us improve IMOS:

    1. Check existing issues: GitHub Issues

    4. Test with minimal case: Isolate the issue to its simplest form

    Version Support Policy

    Support Lifecycle

    • Major versions (e.g., v1.x.x): Supported for 2 years after release
    • Minor versions (e.g., v0.x.x): Supported until next major release
    • Patch versions (e.g., v0.1.x): Supported until next minor release

    Security Updates

    • Critical security issues: Patched within 48 hours for supported versions
    • Important security issues: Patched within 1 week for supported versions
    • Minor security issues: Included in next scheduled release

    Deprecation Policy

    • Features marked deprecated: Removed in next major version
    • Minimum 6 months notice: For any breaking changes
    • Migration guides: Provided for all breaking changes
    • Backward compatibility: Maintained within major versions
